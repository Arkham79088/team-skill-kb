## 🥇 Render.com - 最推荐

### ✅ 优势

1. **完全免费** - Web Service 永久免费
2. **一键部署** - 识别 `render.yaml` 自动配置
3. **内置数据库** - PostgreSQL 1GB 免费 90 天
4. **自动 HTTPS** - 免费 SSL 证书
5. **持续部署** - Git 推送自动更新

### ⚠️ 限制

1. **休眠机制** - 15 分钟无访问自动休眠
   - 首次访问需等待 30-50 秒唤醒
   - 可通过定期访问规避（如 UptimeRobot）
   
2. **数据库有效期** - 90 天后需手动续期或升级
   - 可导出后重新创建免费库
   - 或升级到付费版 $7/月

3. **性能限制** - 512MB RAM
   - 适合 5-10 人小团队
   - 不适合高并发场景

### 💰 成本

- **首 3 个月**: $0/月
- **3 个月后**: 
  - 选项 A: 继续使用免费 Web Service + 重建免费 DB = $0/月
  - 选项 B: 升级付费 DB = $7/月
  - 选项 C: 升级不休眠 Web Service = $7/月

### 🎯 适合人群

- ✅ 初创团队验证想法
- ✅ 内部工具（不介意偶尔唤醒）
- ✅ 学习/演示项目
- ✅ 预算有限的个人开发者

### 🚀 部署时间

**30 分钟** - 包含 GitHub 上传和 Render 配置

## 🥈 Vercel + Supabase - 技术流首选

### ✅ 优势

1. **极致性能** - Vercel 全球 CDN 加速
2. **永久免费** - 两个平台都有永久免费层
3. **无休眠** - 服务持续运行
4. **开发体验好** - 热更新、预览部署

### ⚠️ 限制

1. **架构复杂** - 需要拆分前后端
   - 前端：Vercel 静态托管
   - 后端：Vercel Serverless Functions
   - 数据库：Supabase PostgreSQL

2. **函数限制** - Serverless 超时 10 秒
   - 复杂查询可能超时
   - 需要优化 API 设计

3. **配置繁琐** - 多个平台切换

### 💰 成本

- **Vercel**: $0/月（免费额度内）
  - 带宽：100GB/月
  - Serverless: 100GB-Hrs/月
  
- **Supabase**: $0/月
  - 数据库：500MB
  - API 调用：50,000/日
  
- **总计**: $0/月（初期完全够用）

### 🎯 适合人群

- ✅ 有全栈开发经验
- ✅ 追求高性能
- ✅ 长期运营项目
- ✅ 需要全球访问速度

### 🚀 部署时间

**1-2 小时** - 需要配置多个服务

## ❓ 常见问题

### Q1: Render 数据库 90 天后数据会丢吗？

**A:** 不会立即丢失，但会过期。建议：
1. 第 85 天导出备份
2. 创建新数据库
3. 导入数据
4. 更新连接字符串

或者升级到付费版 $7/月。

### Q2: 休眠唤醒真的需要 50 秒吗？

**A:** 是的，但只在首次访问时。后续访问正常。
可以通过以下方式避免：
- 使用 UptimeRobot 定期访问
- 分享给团队成员增加访问频率
- 升级到付费版

### Q3: 三个平台都能绑定自定义域名吗？

**A:** 都可以！
- Render: 免费，自动 HTTPS
- Vercel: 免费，自动 HTTPS
- Railway: 免费，自动 HTTPS

### Q4: 如果用户超过 10 人会怎样？

**A:** 免费计划可能不够用：
- Render: 考虑升级不休眠版 ($7/月)
- Vercel: 通常够用，超限自动升级
- Railway: 按量付费，约$15-20/月

### Q5: 能从 Render 迁移到 Railway 吗？

**A:** 完全可以！
1. 在 Railway 创建新项目
2. 连接同一个 GitHub 仓库
3. 使用迁移工具导出数据
4. 导入到新数据库
5. 切换 DNS 指向

整个过程 1-2 小时即可完成。
