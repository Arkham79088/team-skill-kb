# 📋 云部署检查清单

> ✅ 按顺序完成每一项，确保部署成功

---

## 🎯 阶段一：部署前准备（10 分钟）

### 账号准备

- [ ] GitHub 账号已注册
- [ ] Railway 账号已注册（使用 GitHub 登录）
- [ ] 已安装 Node.js 和 npm
- [ ] 已安装 Railway CLI：`npm install -g @railway/cli`

### 代码准备

- [ ] 所有文件已保存到本地
- [ ] 已创建 Git 仓库并初始化
- [ ] 已测试本地运行（推荐）

### 安全准备

- [ ] 准备好强密码（至少 12 位）
- [ ] 生成 JWT 密钥：`openssl rand -hex 32`
- [ ] 准备好自定义域名（可选）

---

## 🚀 阶段二：执行部署（15 分钟）

### 方式 A：一键部署脚本（推荐）

- [ ] 进入项目目录：`cd team-skill-knowledge-base`
- [ ] 运行部署脚本：`./deploy-to-railway.sh`
- [ ] 按提示输入管理员密码
- [ ] 等待部署完成
- [ ] 记录后端和前端 URL

### 方式 B：手动部署

- [ ] 登录 Railway：`railway login`
- [ ] 初始化项目：`railway init`
- [ ] 设置环境变量：
  - [ ] `ADMIN_PASSWORD`
  - [ ] `JWT_SECRET`
  - [ ] `CORS_ORIGINS`
- [ ] 添加数据库：`railway add postgresql`
- [ ] 部署后端：`cd backend && railway up`
- [ ] 部署前端：`cd frontend && railway up`

---

## 🔒 阶段三：安全配置（5 分钟）

### Railway 平台安全

- [ ] 启用 Authentication：Settings → Authentication → Require authentication
- [ ] 设置访问密码或邀请码
- [ ] 限制团队成员访问权限

### 应用安全

- [ ] 验证 CORS 配置正确
- [ ] 确认 DEBUG=false
- [ ] 测试管理员登录
- [ ] 验证 API Token 生成正常

---

## 🌐 阶段四：域名配置（可选，10 分钟）

### 自定义域名

- [ ] 购买域名（如 yourteam.com）
- [ ] 在 Railway 添加域名：`railway domain set skills.yourteam.com`
- [ ] 在域名服务商添加 CNAME 记录
- [ ] 等待 DNS 生效（最多 48 小时）
- [ ] 配置 HTTPS（Railway 自动提供）

## 🆘 故障排查清单

如果遇到问题，检查以下项：

### 部署失败

- [ ] Railway CLI 版本是否最新
- [ ] 网络连接是否正常
- [ ] 查看构建日志：`railway logs`

### 无法访问

- [ ] 服务是否正在运行
- [ ] 防火墙/安全组配置
- [ ] DNS 是否生效

### 数据库错误

- [ ] DATABASE_URL 是否正确
- [ ] 数据库插件是否已添加
- [ ] 连接数是否超限

### 前端空白

- [ ] 浏览器控制台是否有错误
- [ ] API 地址配置是否正确
- [ ] CORS 配置是否匹配
